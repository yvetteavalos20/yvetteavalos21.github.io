What the URL?

Web design is filled with elements that just seem to fade onto the background. We use them over and over again to build sites, but never really give them a whole lot of thought. URLs certainly fit into the category and if there is any one part of web design that is sadly overlooked it is learning exactly what a URL is.URL-Uniform Resource Locator.

Although we tend to think of it as a single entitiy that is used for links or web addresses they are actually made up of several parts and the syntax used within them helps servers determine what type of file you are looking for, where to find the pile and can also pass along several optional parameters to the server.

Http: is known as the protocol, it tells the browser to use HTTP as its protocol. The two forward slashes are for correct syntax only. The resource name is identified by a series of domains separated by periods between them. The www is the subdomain. The domain name, this is the name that the browser uses to check with the web server to see if the site exists or not. Then .com is the top level domain, this is the name server that your browser will use to resolve the location of the site that you are requesting .

Common top level domains are; com, org, and edu althought this is not all of them. You do not even need to type the www syntax anymore. Subdomain-It is just a folder that your web content is located in on your web server. Most web servers serve multiple types of content ans subdomains allow servers to identify unique sites or unique areas of content. Www has evolved as the default subdomain for most web content . There are many subdomains besides www, such as ftp or video. The server admin can create create as many subdomains as they wish. A URL should also specify the path on the server to the requested file and which port on the server you should connect to. Every site has a default page, often referred to as the home page of the site, in reality the server can set any page it wishes to as the default page.
