Getting to know HTTP

HTTP-HyperText Transfer Protocol. It is the standard protocol for transferring resources on the web. Remember a protocol is simply a standarized format for transmitting data between two devices. Your browser is an HTTP client while the web server that hosts the requested site is an HTTP server. Now the browser sends an HTTP request explaining which resources that it requires.

Prompts the server to send an HTTP response which in most cases includes the resources that were requested. There are other protocols commonly used to transfer data and most differ based on the type of data requested which data compression to use if any and how the client and server communicate to each other that information has been requested and recieved.

Other common protocols that you may encounter include FTP, SMTP, and RTP. FTP-File Transfer Protocol and is the most widely used protocol for sending and receiving files over the web. HTTP is what we call a one-way stateless protocol and this one way nature make sending and recieving large files over HTTP very difficult. FTP connects to a file server instead of a web server and it is ideal for transmitting larger files, because of this designers regularly use FTP connections when uploading and downloading files from their site.

SMTP-Simple Mail Transfer Protocol is the internet standard protocol for email transactions across the Web. In most email clients, SMTP is used to send email while POP, or Post Office Protocol is used for receiving mail. RTP-Real-Time Transfer Protocol is a standard protocol for delievering audio or video over the internet. It is most commonly used in voice over IP and chat applications.

HTTP is a stateless protocol which means that once a request has been sent and received it is discarded and forgotten by both the browser and the web server. The stateless nature is not without its downside. Nothing about the request is remembered. So tasks such as storing persistent data logins and browsing sessions or a passing product information through a shopping cart must be performed by means other than the request itself. This is done through several different methods, such as using cookies, which are small text files that the page writes on the client machine. You might also see HTTPS which is a variation of HTTP which stands for HyperText Transfer Protocol. Secure and it is simply a layering of a security protocol along with HTTP.Th is allows you to send secure requests from clients to server and is widely used any time security is needed such as e-commerce site.
